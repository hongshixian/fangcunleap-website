# 方寸跃迁产品线

## Agent Runtime 安全

Agent Runtime 安全面向 AI Agent 从"生成内容"到"执行动作"，再到"长期运行"后的新型安全挑战，重点关注 Agent 在真实系统环境中运行时产生的身份权限风险、失控行为风险、供应链风险。

目前平台已形成较完整的运行时安全能力框架，主要包括以下几个方向：

### （1）Agent IAM：身份认证与权限管理

平台已实现面向 Agent 的身份认证与权限管理能力，为不同 Agent、不同任务、不同工具调用建立独立身份与权限边界。

通过 Agent 身份标识、任务级授权、工具级权限控制、最小权限策略等机制，避免 Agent 在运行过程中无限制访问文件、网络、系统命令、浏览器、邮箱、数据库等敏感资源。

该能力可以支持对 Agent 的权限分级、调用审计和越权检测，使每一个 Agent 的行为都能够被归属到明确身份，并根据任务场景授予必要权限，降低因提示词注入、恶意工具、错误规划或模型幻觉导致的越权操作风险。

### （2）运行时观测与系统层副作用监控

平台已实现对 Agent 运行过程的系统层观测能力，能够采集和分析 Agent 在执行过程中的关键行为轨迹，包括文件读写、网络访问、进程执行、命令调用、工具调用、MCP/Skill 调用、浏览器操作等系统层副作用。

在此基础上，平台能够形成完整的 Agent 轨迹，对一次用户请求背后触发的多轮 LLM 调用、多次工具调用和系统副作用进行还原。

通过对 Agent 轨迹的持续数据挖掘，可以进一步识别典型攻击链路、异常行为模式、高风险工具组合和潜在失控路径，为智能体安全威胁建模提供真实运行数据支撑。

该能力的价值不仅在于"看见 Agent 做了什么"，还在于从运行数据中总结风险模式，形成可复用的安全规则、检测特征和威胁模型。

### （3）运行时护栏：低时延安全检测与阻断

平台已实现运行时护栏能力，可在 Agent 执行动作前或工具调用前进行低时延安全检测，当前检测时延可达到约 8ms 级别，能够满足 Agent 运行时实时检测与在线拦截的需求。

运行时护栏主要用于识别高风险操作、越权调用、异常文件访问、敏感数据外发、可疑命令执行、恶意网络请求、危险工具链组合等行为，并根据策略进行放行、告警、阻断或降级处理。

该能力使安全机制不再停留在离线审计阶段，而是能够嵌入 Agent 执行链路，在风险真正造成影响前进行拦截。

### （4）Skill / MCP 安全检测器

平台已实现针对 Skill 和 MCP 生态的安全检测能力，能够对 Agent 可调用的外部能力进行安全分析和风险识别。

随着 Agent 通过 Skill、MCP Server、插件、工具脚本等方式接入外部系统，工具供应链逐渐成为智能体安全中的关键风险入口。

Skill / MCP 安全检测器主要面向工具描述、调用接口、权限声明、代码逻辑、外部依赖和潜在恶意行为进行分析，识别其中可能存在的隐蔽指令、过度权限、敏感数据访问、危险系统调用、网络外传、命令执行、持久化行为等风险。

通过在工具接入前进行检测，可以降低恶意 Skill、恶意 MCP Server 或高风险插件被 Agent 调用后造成系统破坏、数据泄漏或行为滥用的可能性。

### （5）Steward Agent：中控监督与多 Agent 管理

平台已实现 Steward Agent 中控智能体能力，用于对系统中运行的多个 Agent 进行统一监督、调度、管理和审计。

Steward Agent 作为上层治理智能体，负责观察各类 Agent 的运行状态、任务目标、权限边界、工具调用行为和异常风险信号，并在必要时进行干预。

该能力可用于多 Agent 场景下的集中式安全治理，包括任务分配、权限协调、风险告警、异常终止、策略更新和行为审计。

相比单点 Guard 机制，Steward Agent 更强调全局视角和跨 Agent 管理能力，能够发现单个 Agent 局部行为中难以识别的协同风险、链式风险和长期累积风险。

### 总结

总体来看，Agent Runtime 安全平台已初步形成"Agent IAM + Runtime Observation + Runtime Guard + Skill/MCP Scanner + Steward Agent"的完整技术闭环，覆盖 Agent 运行前、运行中和运行后的关键安全节点，为智能体进入真实业务系统提供基础安全底座。

## 大模型安全评测

### （1）模型安全 Red-team 测试

面向前沿大模型开展系统化安全测试与人工对抗验证，覆盖网络安全、生化安全、失控风险、模型欺骗、智能体行为等前沿安全维度，识别模型在复杂高风险场景下的潜在安全漏洞与能力边界。

### （2）Multi-Agent 自动化攻击

通过多智能体协同对最新模型发起复杂、多阶段攻击，挖掘单点测试难以发现的深层漏洞；同时联动攻击策略、风险分析与防御机制，形成“攻—防—评”闭环。

### （3）自动化红队评测平台

自动生成攻击用例并持续执行规模化红队对抗测试，支持多类风险场景覆盖、测试任务编排、结果复现、风险归因与评测报告输出，提升模型安全评测的效率、覆盖度与可持续性。
